1. Pendahuluan

Setelah memahami Padding, Margin, dan Border (Box Model), sekarang kita masuk ke bagaimana kotak-kotak tersebut saling menyusun diri di layar (Display) dan bagaimana menghitung total ukuran kotak tersebut (Box-Sizing).

Materi ini adalah kunci untuk mencegah layout Anda "bocor" atau terpotong tanpa alasan yang jelas.

2. Jenis-jenis Properti Display

Di HTML, setiap elemen secara *default* sudah memiliki nilai display. Contoh: <div> adalah block dan <span> adalah inline. Kita bisa memanipulasinya di CSS:

Display Karakteristik & Perilaku Elemen Default HTML
block Memakan ukuran penuh secara mendatar (100% dari parent). Selalu memulai baris baru dari atas ke bawah. Nilai width dan height berpengaruh. <div>, <p>, <h1>, <ul>
inline Hanya memakan lebar sebesar kontennya (seperti teks di dalam kalimat). Elemen tidak pindah baris walau bersebelahan. Perhatian: width, height, serta margin/padding vertikal tidak berlaku! <span>, <a>, <strong>
inline-block Perpaduan ajaib keduanya. Mengalir secara mendatar sejajar seperti inline, tapi menerima properti width, height, margin, dan padding seperti block. <img>, <button>
none Menghilangkan elemen secara total dari halaman. Elemennya seperti musnah dan ruangnya akan diisi elemen lain. <script>, <style>

Contoh Implementasi HTML & CSS

Perhatikan bagaimana mengubah properti display dapat mengubah perilaku elemen HTML:

<style>
  .span-jd-block {
    display: block;
    background-color: #bfdbfe;
    padding: 10px;
  }
  .div-jd-inline {
    display: inline;
    background-color: #fca5a5;
  }
</style>

<!-- <span> aslinya inline, tapi dipaksa jadi block (memenuhi layar dan pindah ke baris baru) -->
<span class="span-jd-block">Saya adalah span yang berperilaku seperti block.</span>

<!-- <div> aslinya block, tapi dipaksa jadi inline (hanya sebesar bungkus teksnya) -->
<div class="div-jd-inline">Div pertama inline.</div>
<div class="div-jd-inline">Div kedua bersebelahan.</div>

4. Masalah klasik: Box Sizing

Pernahkah Anda mengatur elemen menjadi width: 100% tapi justru kotaknya "tumpah" (melebihi lebar layarnya) saat Anda memberikan Padding atau Border?

Ini disebabkan oleh cara CSS lawas menghitung total lebar elemen.

Terdapat dua jenis Box Sizing:

  • content-box (Default):
    Jika Anda menulis: width: 200px; padding: 20px; border: 5px;, CSS akan membuang ukuran padding dan border di luar batas width awal.
    Lebar Total = 200 (konten) + 40 (padding) + 10 (border) = 250px. Akibatnya elemen membengkak dari rencana.

  • border-box:
    Jika Anda mendeklarasikan box-sizing: border-box;, maka Padding dan Border akan memakan ruangan tersebut ke dalam tanpa merubah ukuran lebar elemen utamanya.
    Lebar Total = Tetap 200px (Ruang konten tersisa otomatis dikurangi menjadi 150px).

Contoh Implementasi HTML & CSS

Bukti bagaimana border-box menyelamatkan tata letak Anda:

<style>
  .box-bawaan {
    box-sizing: content-box; /* Bawaan default CSS */
    width: 200px;
    padding: 25px;
    border: 5px solid red;  
    /* Hasil lebar di layar: 200 + 50 (padding kiri-kanan) + 10 (border) = 260px */
  }

  .box-modern {
    box-sizing: border-box;
    width: 200px;
    padding: 25px;
    border: 5px solid green;
    /* Hasil lebar di layar: TETAP 200px secara konsisten */
  }
</style>

<div class="box-bawaan">Kotak ini bocor (260px)</div>
<div class="box-modern">Kotak ini rapi (200px)</div>

6. Best Practices Web Modern

Karena konsep content-box sangat sering mematahkan *layout*, merupakan Standar Emas (*Global Reset*) di era modern untuk memaksa seluruh elemen menggunakan border-box.

Selalu tuliskan snippet ini di baris pertama file CSS Anda:

/* Terapkan border-box ke semua elemen */
*,
*::before,
*::after {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
}

Setelah menerapkan kode di atas, saat Anda menulis width: 100%;, maka ia akan benar-benar mencakup padding dan border tanpa merusak tata letak layar Anda.

7. Latihan Praktik: Menata Menu Navigasi

Elemen daftar atau list (`<li>`) secara default adalah block sehingga ia akan berbaris ke bawah. Jadikan struktur ini sebagai menu navigasi yang berderet ke samping dengan trik inline-block. 

<style>
  * {
    box-sizing: border-box;
  }
  
  .nav-menu {
    background-color: #1e293b;
    padding: 15px;
    list-style: none; /* Hilangkan titik-titik pada list */
  }

  .nav-item {
    display: inline-block; /* Ubah perilaku dari "block" ke "inline-block" */
    margin-right: 20px;
  }

  .nav-item a {
    color: white;
    text-decoration: none;
    padding: 10px 15px;
    border-radius: 6px;
  }

  .nav-item a:hover {
    background-color: #3b82f6;
  }
</style>

<ul class="nav-menu">
  <li class="nav-item"><a href="#">Beranda</a></li>
  <li class="nav-item"><a href="#">Produk</a></li>
  <li class="nav-item"><a href="#">Tentang Kami</a></li>
</ul>

Silakan simpan dan jalankan di format HTML Anda. Anda akan melihat sebuah *navbar* sederhana dan proporsional tanpa terpotong!

9. Ringkasan Akhir

  • Block memakan garis lurus penuh dan bisa diatur *width / height*.
  • Inline hanya mengambil area sebesar panjang teks, dan width/height tidak bekerja padanya.
  • Inline-Block bisa berjajar menyamping seperti Inline, tetapi bisa diberi gaya ukuran seperti Block.
  • Hampir seluruh *web developer* saat ini menggunakan reset * { box-sizing: border-box; } demi kewarasan perhitungan layar (mencegah padding membuat elemen bocor melebar).
